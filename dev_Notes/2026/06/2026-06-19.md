## 작업 내용

### ROS 로그 분석 및 이슈 원인 파악
- 수신한 ROS 로그 분석을 통해 전처리기 Controller 내부 이슈 원인 확인
- 전처리기에서 ReadyToLoad 메시지 수신 시 Job 초기화를 진행하는데, 초기화 과정에서 변수가 정상적으로 공유되지 않는 것으로 확인
- 변수 공유 실패로 인해 Job이 내부적으로 이미 존재하는 상태로 인식되어 정상 시퀀스 진행 불가 상태 발생

### 전처리기 Controller 코드 수정
- ReadyToLoad 메시지 수신 후 Job 초기화 시 변수가 정확히 공유되도록 코드 수정
- 설비 상태 표시 변경 시점을 초기화 시점으로 이동하여 개선
- 수정된 프로그램 빌드 후 현장 전달

## 배운 점 / 느낀 점

- 로그만으로 원인을 특정하기 어려운 경우 추가 로그 수집 → 분석 → 수정의 단계적 접근이 효과적임
