## 오늘 한 일
- 전처리기 Controller 개선 작업 (ROS IP:PORT 연동 및 설비별 서버 관리)

## 작업 사유
- ROS에서 수신한 설비별 IP:PORT 정보를 UI에서 확인 및 수정할 수 있어야 하며, 해당 정보 기반으로 설비별 서버를 동적으로 운용할 필요가 있어 개선 착수

## 작업 내용

### 전처리기 Controller - IP:PORT 목록 UI 표시 및 수정
- ROS에서 수신한 설비별 IP:PORT 목록을 UI에 표시
- 수정된 데이터를 수신 시 목록에 즉시 반영되도록 처리
- 목록은 별도 저장하지 않고, ROS로부터 데이터를 받기 전까지는 설비 없음 상태로 동작하도록 구현

### 전처리기 Controller - 설비별 서버 오픈 기능 개선
- ROS에서 받은 IP:PORT 기준으로 설비별 서버를 개별 오픈할 수 있도록 개선
- IP:PORT 데이터 수정 시 서버를 OFF → ON 재기동하여 변경된 정보로 서버가 열리도록 처리

## 배운 점 / 느낀 점
- 설비 정보를 저장 없이 수신 기반으로만 관리하면 상태 동기화가 단순해지지만, 재기동 시 초기화 처리를 명확히 해야 함
