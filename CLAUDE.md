# 프로젝트 개요
개인 일일 개발 노트 및 주간업무일지 자동화 시스템.
노트는 한국어로 작성하며 dev_Notes/YYYY/MM/YYYY-MM-DD.md 형식으로 저장.

# 디렉토리 구조
- dev_Notes/YYYY/MM/ : 일일 개발 노트 (.md 파일만, .json 없음)
- weekly_reports/YYYY/MM/ : 주간업무일지 .txt 파일
  예) weekly_reports/2026/05/5_1_Weekly_Journal.txt, weekly_reports/2026/05/5_2_Weekly_Journal.txt

# 규칙
- 파일 생성/수정 완료 후 커밋 여부를 절대 묻지 말 것
- .json 파일 생성 금지
- 커밋 메시지 형식: YYYY-MM-DD Todo_Update (파일 1개일 때), YYYY-MM-DD All_Todo_Update (파일 2개 이상일 때)
- 주간업무일지 파일 이미 존재하면 덮어쓰기
- 코드 및 답변은 한국어 사용
- 일일 개발 노트 작성 완료 후 반드시 VSCode로 자동으로 열어줘 (code 파일경로 명령어 사용)
- 주간업무일지 작성 완료 후 반드시 notepad++로 자동으로 열어줘. notepad++가 없으면 메모장(notepad)으로 열어줘
- 작업 내용 소제목은 간결하게 작성할 것 (예: LAS-UI 오류 확인, DB 오류 개선)
- 향후 계획에 체크박스( [ ] ) 사용하지 말 것
- 업체명 등 고유명사는 사용자가 입력한 그대로 사용할 것
- 주간업무일지 작성 시 전주 계획 항목은 반드시 저번 주 주간업무일지 파일을 먼저 읽어서 향후 계획 항목을 확인한 후, 실제 이행 여부를 개발 노트와 대조해서 작성할 것
